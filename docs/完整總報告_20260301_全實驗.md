## 二、Full-model（重點）

### 2.1 ViT FP32（train+bwd+ft）
以 `tmp_20260301_vit_fp32_math512_ab.csv` 為主：
- 對 SDPA-Math：ELSA 中位速率 `1.7247x`，VRAM 比 `0.2261x`。
- 對 SDPA-Mem：ELSA 中位速率 `1.0888x`，VRAM 比 `1.0458x`。

### 2.2 ViT FP32 低記憶體選項
以 `tmp_20260301_vit_fp32_memsaveout0_ab.csv`：
- 對 SDPA-Mem：VRAM 比可壓至 `~0.9998x`，但速率變為 `~0.9356x`。
- 結論：可提供速度/VRAM 交換，預設仍建議速度優先。

### 2.3 Swin FP32
以 `tmp_20260228_swin_fp32fp16_regress.csv`：
- ELSA-Swin-FP32 對原生 Swin-Window 中位速率 `1.0353x`，VRAM 比 `0.9494x`。
- 結論：Swin FP32 在目前主線為小幅穩定優勢。

### 2.4 ViT FP16
以 `tmp_20260228_fp16_vit_img1536_fair4rep.csv`：
- 對 SDPA-Flash 中位速率 `1.0003x`（幾乎打平），VRAM 比 `1.0003x`。
- 結論：FP16 目前定位為與 FA/Flash 系列競爭下的可接受路徑，長序列場景另有 ELSA-stream 優勢（見 LLM 區）。

## 三、Backward 路徑修復與防呆
- 問題路徑：`ELSA_TRITON_FP32_TRAIN_BWD=triton` 在此 stack 可嚴重退化（`~0.02x`，見 `tmp_20260301_vit_fp32_bwd_triton_ab.csv`）。
- 修復策略：加入 guard，預設阻擋該路徑；只有 `ELSA_TRITON_ALLOW_UNSTABLE_PATHS=1` 才允許。
- 修復後：`tmp_20260301_vit_fp32_bwd_triton_guard.csv` 恢復正常比較趨勢（對 SDPA-Math 約 `1.57x`）。

## 四、精度與一致性
- 訓練 loss 曲線（ViT FP32）對齊：`abs loss diff` 中位為 `0`，p95 約 `1.18e-7`。
- LLM perplexity（ELSA vs ELSA-stream）：差異 `7.48e-06`，可視為一致。
- 下游 top-1 在目前樣本設定可重現；請固定 seed 與資料切分後解讀（樣本數有限）。

## 五、結論
1. 本版可作為乾淨 release 候選：主線/實驗路徑已分離，且主要結果可重現。 2. ViT FP32（train+bwd+ft）為主戰力：對 SDPA-Math 全面優勢、對 SDPA-Mem 多數優勢且 VRAM 接近。 3. Swin FP32 已達可接受優勢；ViT FP16 已接近主流 fast baseline，長序列與 streaming 場景仍具亮點。 4. 已知不穩定路徑已被 guard，不會污染主線結論。
